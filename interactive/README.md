# Interactive articles

Source and durable generated snapshots for the new root interactive site.

Current rules:

- Canonical sources live in `interactive\source\YYYY\slug.article.md`.
- Durable rendered snapshots live in `interactive\generated\YYYY\slug\`. These are copied to `_site\YYYY\slug\` by the site build, then patched with index navigation and a related-article footer.
- Compact semantic metadata lives in `interactive\article-index.json`. Update it whenever an interactive article is added or materially rewritten.
- Shared generation assets live with the generator skill in `.agents\skills\interactive-article-generator\assets\`.
- Generated public output is written to `_site\`; the classic Jekyll blog is built under `_site\classic\`.
- Generated snapshots are source artifacts, but prose and structure should still be changed in the `.article.md` source first and regenerated from the skill contract.
- The generated root index uses relative links, newest-first cards, collapsed search, collapsed theme cards, and batched archive reveal for larger article counts.
- Public article pages are patched so the first main card is open by default and later cards start collapsed.

The root landing page, `search.json`, shared assets, backlinks, and related recommendations are generated by:

```powershell
python tools\build_site.py
```

The generator is deterministic and uses no LLM calls. Agent-maintained metadata in `interactive\article-index.json` provides short summaries, labels, and theme assignments so recommendations and topic groupings can stay clever without spending tokens during every CI build.
